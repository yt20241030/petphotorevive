## 1. 目标与范围

**唯一目标:7 天内能上线收到第一笔钱。** 只做一件事,别的一律不做:

> 上传老宠物照片 → AI 修复清晰 → 带水印预览 → 付费 → 下载高清无水印。

- 内容主题:**宠物**老照片修复(不涉及人脸)。
- 界面语言:**先英文**(目标客户是海外 Rainbow Bridge / 宠物悼念人群)。
- 不做:App、PWA、手机专属界面、用户系统、后台管理、上色、划痕修复等。这些都往后放。

## 2. 技术栈(已定,不要自行更换)

- **框架**:Next.js(App Router)+ TypeScript。标准 Web 应用,桌面优先。
- **响应式要求**:不做手机专属功能,但页面**必须在手机浏览器能正常打开、能上传、能付款**(目标客户大概率用手机)。用 Next.js 默认响应式即可,别为手机加额外工程,但别让它在手机上塌。
- **部署**:Vercel(**Pro 计划**,商业用途必须 Pro,免费 Hobby 禁止收款)。
- **CDN / 防护**:前置 **Cloudflare**(挡 DDoS、缓存);**图片走 Cloudflare R2**(出站流量免费),避免每张修复图都吃 Vercel 计费流量。R2 可在核心功能跑通后接入,别因此阻塞首次上线。
- **修复模型**:**Replicate** 的 `nightmareai/real-esrgan`,**仅服务端调用**。
  - `face_enhance` 参数**必须设为 false**。原因:我们修的是宠物、不需要人脸增强;而该选项依赖 GFPGAN,GFPGAN 带 CC-BY-NC(DFDNet)非商用依赖。关掉它,才保证只用干净可商用的 Real-ESRGAN(BSD)。
  - `scale` 默认 2(可配置);老照片 2x 通常够,4x 偏大偏贵。
- **收款**:**Dodo Payments** 的 Checkout + Webhook。**先用测试模式(test mode)开发**,Dodo 审核通过后再切正式模式,不要等审核。

> Replicate、Dodo 的具体 API 形态请以官方文档为准(`replicate` 官方 SDK;Dodo 见 docs.dodopayments.com 的 Checkout 与 Webhooks 章节),不要凭记忆硬编码接口。

## 3. 产品流程(把这条流水线实现出来)

1. **落地页(英文,单页)**:简短价值主张 + 上传控件 + 一个前后对比示例。
2. 用户上传老宠物照片 → 前端把图**发到服务端**(不在前端直接碰模型)。
3. 服务端调用 Replicate `nightmareai/real-esrgan`(`face_enhance=false`, `scale=2`),拿回高清修复图。
4. 服务端生成两份:
   - **预览图**:降分辨率 + 打水印,用于展示;
   - **干净高清原图**:存到私有存储(R2 优先,私有桶),按 `jobId` 归档,**绝不暴露给前端**。
5. 页面展示**前后对比(滑块)** + 带水印预览 + 按钮:`Download HD (no watermark) — $X`。
6. 点击 → 跳 Dodo Checkout(hosted / overlay),携带该 `jobId`。
7. **Dodo Webhook 回调** → 服务端**校验签名** → 标记该 `jobId` 已付 → 生成一次性 / 带签名的下载链接,放出干净高清原图。

## 4. 必须在代码里焊死的红线

1. **密钥只在服务端环境变量**,绝不进前端 bundle、绝不进 URL、绝不硬编码。
2. **成本闸防刷**(每张 Replicate 调用都花钱,必须防):
   - 按 IP / 会话限制**免费预览次数**(默认每天 3–5 次,可配置);
   - 上传图片按内容 **hash 去重缓存**:同一张图已处理过就复用结果,不重复调 Replicate;
   - **每个 job 只调一次** Replicate;付费下载复用同一张,不重跑;
   - 校验上传:仅 jpg/png/webp,单张 ≤ ~10MB,拒绝超大图;
   - 打开 **Vercel spend management**(用量告警 + 到顶暂停)。
3. **预览必打水印 + 降分辨率;干净高清仅在 Webhook 校验付款成功后释放。** 严禁凭前端跳转就放行——必须以服务端验签的 Webhook 事件为准。
4. **只用 Real-ESRGAN,`face_enhance=false`。** 不引入其它模型。
5. 下载链接**一次性 / 短时效**,防止被转发白嫖。

## 5. 密钥与账号(用环境变量,别硬编码)

创建 `.env.local`(本地)+ 在 Vercel 里配同名环境变量。需要时,明确告诉创始人"去哪拿这个 key、贴到哪个位置",并提醒**不要把 key 发到别处**。

- `REPLICATE_API_TOKEN` —— Replicate 后台获取。
- `DODO_API_KEY` —— Dodo 后台(先用 test key)。
- `DODO_WEBHOOK_SECRET` —— Dodo Webhook 设置里获取,用于验签。
- (接入后)`R2_*` —— Cloudflare R2 的访问凭证。

> 创始人是中国大陆个人,通过 Dodo 收款、经 Payoneer 提现到国内卡。这部分不涉及代码,你不用管;你只需保证收款用 Dodo、把钱正确记账到订单即可。

## 6. 建站顺序(每步都要能跑、能测;别一口气写完再测)

1. **脚手架 + 先部署一个"Hello"到 Vercel**,确认"代码→GitHub→Vercel→线上 URL"整条链路通。**尽早拿到这个线上网址**——它正好可以去填 Dodo / Payoneer 的"网站网址"栏。
2. **上传 → Replicate 修复 → 显示修复图**(先不做支付)。让核心"变清晰"的效果先跑通,用真实老宠物照片测几张,确认出图值得让人掏钱。
3. **加水印预览 + 把干净高清藏到服务端**(未付款拿不到)。
4. **接 Dodo Checkout + Webhook(测试模式)**,验签后才放行下载。
5. **上成本闸**(限次、去重缓存、上传校验)+ 开 Vercel 用量上限。
6. **接 Cloudflare + R2**(把图片流量挪走)。
7. Dodo 审核通过后**切正式模式**,真实收钱。

## 8. 设计方向

- 干净、温暖、可信;单页,**前后对比滑块是主角**。
